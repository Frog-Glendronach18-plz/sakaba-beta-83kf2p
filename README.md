# 酒場ドラフト戦記

仲間を雇って配置し、俯瞰マップの自動戦闘を勝ち抜くオートバトラー。GitHub Pages で公開・PWA対応（ホーム画面に追加すればオフラインでも遊べる）。

## ファイル構成

```
index.html            エントリーポイント（PWAメタ情報・スクリプト読み込み）
manifest.webmanifest  PWAマニフェスト（アプリ名・アイコン・表示モード）
sw.js                 Service Worker（オフラインキャッシュ）
css/style.css         全スタイル
icons/                PWAアイコン（192/512/maskable/apple-touch）
audio/bgm/            BGM（m4a/AAC）: メニュー / 旅立ち / 戦場(草原・山・雪) / ボス / 宴 / 勝利 / 敗北
                      ※iOS Safariはogg非対応のためAAC(m4a)採用。ファイル名は空白なし小文字。
audio/se/             効果音（今後追加予定。現状は空。SFXはコード生成のWebAudio）
js/                   ゲーム本体（読み込み順に依存関係あり。index.html の順序を維持すること）
  data.js             ゲーム定数・ユニット/特性/イベント定義・戦績記録・バイオーム
  audio.js            WebAudio 効果音（コード生成のSFX）
  bgm.js              BGM(m4a)の再生管理（切替・ループ・音量・ダッキング・ミュート連動）
                      ※音量はGainNode制御。iOS SafariはHTMLAudioElement.volumeを無視するため、
                        volume直接操作に書き換えるとスマホで音量調整・フェード停止が壊れる
  state.js            ゲーム状態・共通ユーティリティ・カード抽選
  shop.js             酒場（募兵ショップ）のカード生成
  enemies.js          敵編成の生成・二つ名・噂
  field.js            戦場ギミック（岩・宝箱）
  team.js             経済・シナジー・部隊管理（購入/合体/売却/配置）
  battle.js           自動戦闘シミュレーション
  rounds.js           ラウンド終了処理・旅立ちイベント・宴・実績
  render.js           盤面/トークン/ショップUIの描画・戦況予測
  fx.js               演出エフェクト
  pwa.js              「ホーム画面に追加」案内・結果の共有（X投稿/コピー）
  resume.js           中断セーブ＆再開（ショップ中に自動保存・タイトルの「続きから」）
  arena.js            闘技場（模擬戦・固定対称フィールド・軍団の出場名簿。既存へのフックは各1行）
  ui.js               タイトル/ヘルプ/チュートリアル・イベント登録・起動（必ず最後に読み込む）
sakaba-draft-senki-v128.html  分割前の旧単一ファイル（バックアップ。公開には不要）
```

各 js ファイルは通常の `<script>`（ES モジュールではない）として順番に読み込まれ、トップレベル宣言をグローバルとして共有する。イベント登録とゲーム起動は最後の `ui.js` が行うため、**読み込み順を変えないこと**。

## ローカルでの動作確認

Service Worker は `file://` では動かないため、ローカルサーバーを使う:

```
python -m http.server 8123
```

→ http://localhost:8123 を開く。

## 開発ツール（ローカル専用・アップロード禁止）

バランス調整用に `dev.html` と `dev/devtools.js` がある。**この2つは GitHub Pages にアップロードしないこと**（万一上げても localhost 以外では無効化されるが、上げない運用が原則）。

使い方: ローカルサーバー起動後 http://localhost:8123/dev.html を開く（`index.html` を取得して開発ツールを注入するだけなので、本体を更新しても dev.html の追従作業は不要）。左下の「🛠 DEVツール」パネルから:

- **チェックポイント**: 現在の状況を保存し、いつでもその場面から再開（ラウンド開始ごとの自動保存つき・localStorage 保存）
- **チート**: ゴールド/体力の設定、任意ラウンドへ移動、部隊枠/鍛錬、任意ユニットを★指定で追加、選択ユニット削除
- **ターボ**: 戦闘を高速化（SPD=8）
- **連戦テスト**: いまの盤面で N 回自動戦闘して勝率と平均被害を計測（毎回自動で巻き戻し。実績は汚さない）
- **戦績ログ**: 全戦闘の結果を自動記録。CSV ダウンロード／ラウンド別勝率の集計表示
- **乱数シード固定**: Math.random を固定して同じ展開を再現（検証用）

## 更新して公開するときの注意

1. コード（js/css/html）を編集したら **`sw.js` の `CACHE_VERSION` を上げる**（例: `sakaba-v1.1.0` → `sakaba-v1.1.1`）。
   これを忘れると、既にインストール済みのユーザーに古いキャッシュが配られ続ける。
2. **BGM（`audio/bgm/`）を追加・差し替えたときは、`sw.js` の `MEDIA` 一覧を更新し `MEDIA_CACHE` を上げる**（例: `sakaba-media-v1` → `v2`）。
   BGMはオフライン再生用に別キャッシュへプリキャッシュされ、コード更新では再DLされない設計。音源を変えたときだけ上げればよい。
   ファイル名に空白がある場合は `MEDIA` では `%20` で表記する（bgm.js 側は `encodeURIComponent` で自動処理）。
3. すべて相対パス（`./`）で書かれているので、GitHub Pages のサブパス（`https://ユーザー名.github.io/リポジトリ名/`）でもそのまま動く。
